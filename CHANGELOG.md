## [0.1.7] - 2025-11-03

### Added
- **version**: Update version to 0.1.7 for release
- **config**: Mandate comprehensive local toolset usage for all AI agents
- **hooks**: Configure automatic loadsession execution for new sessions
- **token-sage**: Complete automatic integration implementation for maximum efficiency

### Changed
- **monitoring**: Enhanced token usage logging optimization
- **build**: Modernize Python packaging with Hatch dynamic versioning
- **test**: Implement 4.9x faster pytest optimization for agricultural robotics
- Clean up CLAUDE_SUBAGENTS_PACKAGE directory structure

### Fixed
- **gitignore**: Add .claude/conversation_config.json and .agent_registry.json to ignore user-specific local configuration
- **docs**: Update version references across documentation and README.md

## [0.1.6] - 2025-10-01

- **session management**: Comprehensive cross-session persistence and state management
- **database**: PostgreSQL integration with Alembic migrations
- **TodosManager**: 12-layer hierarchical task management framework
- **CI/CD**: Automated dependency updates and security monitoring

### Changed
- **architecture**: Multi-tractor fleet coordination with vector clocks
- **testing**: Enhanced test suite expansion to 846 tests
- **documentation**: Comprehensive project documentation and user guides

### Fixed
- **performance**: Optimize database operations and API response times
- **security**: Address dependency vulnerabilities and improve input validation

## Maintaining This Changelog

### Guidelines:
1. **Use [Keep a Changelog](https://keepachangelog.com/) format**
2. **Categorize changes**: Added, Changed, Deprecated, Removed, Fixed, Security
3. **Include dates in YYYY-MM-DD format**
4. **Link GitHub issues and pull requests**
5. **Keep Unreleased section for upcoming changes**

### Archive Strategy:
- **Keep current 6 releases** in main CHANGELOG.md (under ~100KB)
- **Archive older releases** quarterly by moving to CHANGELOG_ARCHIVE/
- **Maintain this structure** for readability and performance

### For Contributors:
1. Add changes to Unreleased section before merging
2. Create new release entry when tagging version
3. Archive old releases periodically (quarterly review)
4. Reference GitHub issues for specific changes
